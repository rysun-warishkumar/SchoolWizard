Make My School – Hosting Guide & Adding a New School

This page explains how to host the school management portal on UTHO Cloud with subdomain demo.makemyschool.com (domain from Hostinger), and how to add a new school later.

Contents

2. Domain (Hostinger) → UTHO

1 In Hostinger (where makemyschool.com is managed): open DNS / Domain management.
2 Add an A record: host demo → value = UTHO server public IP.
3 Wait for DNS (often 5–15 minutes; up to 24–48 hours). Verify with ping demo.makemyschool.com or nslookup demo.makemyschool.com.

3. Server setup (UTHO VPS – Ubuntu)

3.1 Install basics

sudo apt update && sudo apt upgrade -y
sudo apt install -y nginx nodejs npm mysql-server

3.2 MySQL

sudo mysql_secure_installation
sudo mysql -e "CREATE DATABASE schoolwizard CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
sudo mysql -e "CREATE USER 'schoolwizard'@'localhost' IDENTIFIED BY 'YOUR_STRONG_PASSWORD';"
sudo mysql -e "GRANT ALL ON schoolwizard.* TO 'schoolwizard'@'localhost'; FLUSH PRIVILEGES;"

Import schema from the project: mysql -u schoolwizard -p schoolwizard < database/consolidated/00_complete_database.sql

3.3 Backend

Clone or upload the project. In backend/:

npm ci
cp .env.example .env
# Edit .env: DB_*, JWT_SECRET (32+ chars), PORT=5000, CORS_ORIGINS=https://demo.makemyschool.com
npm run build
npm start   # test; then use PM2

Production .env example:

NODE_ENV=production
PORT=5000
DB_HOST=localhost
DB_USER=schoolwizard
DB_PASSWORD=YOUR_STRONG_PASSWORD
DB_NAME=schoolwizard
JWT_SECRET=your-minimum-32-character-secret-here
JWT_EXPIRE=7d
CORS_ORIGINS=https://demo.makemyschool.com

Run with PM2: pm2 start dist/index.js --name make-my-school-api then pm2 save and pm2 startup.

3.4 Frontend & SchoolPortal

Build locally or on server with production API URL:

# Admin frontend
cd frontend
echo "VITE_API_BASE_URL=https://demo.makemyschool.com/api/v1" > .env.production
npm ci && npm run build

# SchoolPortal
cd ../SchoolPortal
echo "VITE_API_BASE_URL=https://demo.makemyschool.com" > .env.production
npm ci && npm run build

Upload frontend/dist/ and SchoolPortal/dist/ to the server (e.g. /var/www/demo/frontend and /var/www/demo/portal).

4. Nginx & SSL

Example Nginx config for demo.makemyschool.com: serve admin at /, portal at /portal, API at /api/.

server {
    listen 80;
    server_name demo.makemyschool.com;
    root /var/www/demo/frontend;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
    location /portal {
        alias /var/www/demo/portal;
        try_files $uri $uri/ /portal/index.html;
    }
    location /api/ {
        proxy_pass http://127.0.0.1:5000/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Enable site, test, reload Nginx. Then install SSL:

sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d demo.makemyschool.com

After SSL, ensure frontend and portal are built with https:// in VITE_API_BASE_URL and backend CORS_ORIGINS uses https://demo.makemyschool.com (no trailing slash).

5. Adding a new school

You can run multiple schools on the same instance (single database, multi-tenant) or run a separate instance per school.

Option A – Same instance (multi-tenant, recommended)

The app has a schools table and school_id on users. To add a school:

  1. Platform admin UI (if you host it): log in and create a new school, set status (e.g. trialing/active), trial end date, storage limit.
  2. Or insert the school and an admin user directly in the database (see Option B).

Option B – Database: add school + admin user

Note: This assumes the SaaS migrations are applied (e.g. schools table and users.school_id). If your schema differs, adjust the SQL.

Run in MySQL (e.g. phpMyAdmin or mysql client):

-- 1) Create school (note the new id, e.g. 2)
INSERT INTO schools (name, status, plan_code, storage_limit_bytes, storage_used_bytes, trial_starts_at, trial_ends_at, created_at)
VALUES (
  'New School Name',
  'trialing',
  'trial',
  21474836480,
  0,
  NOW(),
  DATE_ADD(NOW(), INTERVAL 30 DAY),
  NOW()
);

-- 2) Create admin user for that school (use bcrypt hash for password)
-- Replace 2 with the school id from step 1; replace email/name and password hash
INSERT INTO users (email, password, name, role_id, school_id, is_active)
VALUES (
  'admin@newschool.com',
  '$2a$10$...',   -- e.g. run: node -e "console.log(require('bcryptjs').hashSync('admin123', 10))"
  'Admin Name',
  1,
  2,
  1
);

Users with that school_id will see only their school’s data. You can later change the school’s status (e.g. to active) and trial end date from the platform admin or via SQL.

Option C – Separate instance (new server)

For full isolation per school:

  1. Provision another VPS (e.g. another UTHO server).
  2. Repeat the same setup: MySQL, backend, frontend, portal, Nginx, SSL.
  3. Point a new subdomain (e.g. school2.makemyschool.com) to the new server’s IP.

Use Option A/B when you want one deployment to serve many schools; use Option C when you need separate infrastructure per school.

6. Go-live checklist

  • DNS: demo.makemyschool.com A record → UTHO IP
  • MySQL: database created, schema imported, credentials in backend .env
  • Backend: JWT_SECRET (32+ chars), CORS_ORIGINS, PM2 running
  • Frontend & Portal: built with correct VITE_API_BASE_URL, deployed under Nginx
  • Nginx: SPA fallback (try_files → index.html) for / and /portal; /api/ proxied to Node
  • SSL: Certbot certificate for demo.makemyschool.com
  • Change default admin password after first login
  • Regular MySQL backups (and uploads/S3 if used)
